# ネタ収集プロンプト（neta-dataset-builder MVP）

Claude や ChatGPT に下のプロンプトを貼り付けると、ネタふるい Lite にそのままインポートできる JSON が返ってきます。
`◆◆ ◆◆` で囲った3箇所を書き換えてから使ってください。

出力された JSON は、ネタふるい Lite の「インポート / エクスポート」画面に貼り付けるだけで取り込めます。

---

## プロンプト本文（ここからコピー）

```
あなたはアイデア収集の専門家です。以下の条件でネタ候補を集め、指定のJSON形式だけを出力してください。

## 収集テーマ

◆◆ここにテーマを書く。例：産業医が10分の衛生講話で使える「睡眠と仕事のパフォーマンス」のネタ◆◆

## 件数

◆◆例：20◆◆ 件

## 観点・条件

◆◆例：
- 一般従業員に話して「明日から行動が変わる」具体性があるもの
- 誇張や煽りではなく、根拠のある内容
- ありきたりすぎるもの（早寝早起きしましょう等）は除外
◆◆

## 出力形式（厳守）

次のJSONだけを出力する。前置き・解説・コードブロック記号以外の文章は一切付けない。

{
  "schemaVersion": "1.0.0",
  "datasets": [
    {
      "id": "ds-<テーマの英語スラッグ>",
      "name": "<データセット名（日本語、簡潔に）>",
      "description": "<このデータセットの説明1行>",
      "fields": [
        { "key": "<項目キー>", "label": "<表示名>", "type": "text", "showOnCard": true }
      ]
    }
  ],
  "cards": [
    {
      "id": "card-<連番>",
      "datasetId": "<上のdatasets[0].idと同じ値>",
      "title": "<ネタの見出し。一覧で内容が想像できる具体性で>",
      "summary": "<2〜3行の概要。何が面白い/役立つのかまで書く>",
      "customFields": {
        "<項目キー>": "<値>"
      },
      "provenance": {
        "sourceTitle": "<出典名。不明なら省略>",
        "sourceUrl": "<出典URL。不明なら省略>"
      }
    }
  ]
}

## JSONのルール

- fields は3〜5個。このテーマのネタを比較・判断するのに役立つ項目を自分で設計する
- 全カードの customFields のキーは、fields で定義した key と完全に一致させる
- status フィールドは付けない（仕分けは人間がやる）
- 出典が実在すると確信できる場合のみ provenance を付ける。不確かな出典をでっち上げない
- 似たネタばかりにせず、切り口を分散させる
- JSONとして必ずパース可能であること（末尾カンマ禁止、文字列内の改行は \n）
```

## 使い方の流れ

1. 上のプロンプトの ◆◆ ◆◆ を埋めて、Claude に投げる
2. 返ってきた JSON をコピー
3. ネタふるい Lite →「インポート / エクスポート」→ テキストエリアに貼り付け → インポート
4. 仕分け画面でふるいにかける
5. 件数が足りなければ「同じ形式で、既出と重複しない切り口でさらに◯件」と追撃する

## コツ

- **1回で完璧を狙わない。** 20件出させて15件捨てるくらいの気持ちで量を出させる
- テーマが広すぎると凡庸なネタばかりになる。「睡眠」より「夜勤明けの過ごし方」のように絞る
- 良いカードの傾向が見えてきたら、その傾向を「観点・条件」に書き足して再収集する
- 同じデータセット名で何度インポートしても、カードは同じデータセットに追記される（id が同じカードは重複しない）

## 将来の自動化（CLI化）について

この手動運用で「収集→仕分け」のサイクルが回ると確認できたら、
このプロンプトをそのままシステムプロンプトにして Claude API を呼ぶ CLI（neta-dataset-builder 本体）に昇格させる。
プロンプトの改善はこのファイルで先に検証する。
