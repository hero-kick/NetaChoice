ネタふるい Lite

ネタをあとで仕分ける — 自分用の軽量仕分けPWA

AIや自分で大量に集めたネタ・アイデア・創作素材を、スマホの空き時間に1枚ずつ見て 「残す / 捨てる / 保留」に直感的に仕分けるためのツールです。 保存するためのメモアプリではなく、大量の候補をふるいにかけて、使えるものだけ残すためのアプリです。

コンセプト

  • 軽い — ログイン・サーバー・クラウド不要。完全ローカルで動くPWA
  • 分かりやすい — 画面は4つだけ（ホーム / 仕分け / 一覧 / インポート・エクスポート）
  • スマホで使いやすい — スワイプ操作、ホーム画面に追加して即起動
  • データを入れやすい — AIが作ったJSONを貼り付け、またはファイル選択で取り込み
  • 分類しやすい — 1画面1カード。迷ったら保留、あとで再ふるい
  • 残したものを取り出しやすい — 「残す」だけをJSONやMarkdownで出力

NetaChoice 3ツール構成の中での位置づけ

ネタふるい Lite は「集める → 仕分ける → 形にする」という3工程の真ん中を担当します。

neta-dataset-builder（予定） AI・Web検索でネタ候補を収集し、共通スキーマのJSONを出力
→
ネタふるい Lite（このツール） 人間がカードを1枚ずつ見て仕分ける。AI機能・動画機能は持たない
→
quote-video-factory（予定） 「残す」カードから短尺動画を生成

ツール間のデータ受け渡しはすべて shared/neta-schema.json 準拠のJSONファイルで行います。 DBやサーバーは介在しません。

データの構造

データは2階層だけです。

階層意味例
データセット分類対象のまとまり。カスタム項目の定義を持つミステリートリック、動画ネタ、衛生講話ネタ
カード1つ1つのネタ・候補時計の針で文字を読む謎、夜勤明けの仮眠は90分か20分か

カードの状態は4種類です。

未分類 残す 捨てる 保留

画面と使い方

1. ホーム画面

データセットの一覧と、今日の進捗（見た / 残す / 捨てる）を表示します。データセットの新規作成・編集もここから行います。

2. 仕分け画面

未分類カードを1枚ずつ表示します。操作方法は3通りあります。

操作残す捨てる保留1つ戻る
スワイプ右左上—
ボタン残す捨てる保留↩ 1つ戻る
キーボード（PC）→←↑Z

未分類をすべて仕分け終わると完了画面になり、保留が残っていれば 「保留 N 件を再仕分けする」からそのまま再ふるいに入れます。 保留モード中は「保留のまま」を選ぶとそのカードを飛ばして次に進みます。

3. 一覧画面

仕分け済みカードを状態・データセットで絞り込み、タイトルで検索できます。 カードをタップすると詳細が開き、状態の変更・編集・削除ができます。手動でのカード追加もここから行います。

4. インポート / エクスポート画面

データの出し入れをまとめた画面です。詳細は次のセクションを参照してください。

インポート

2つの方法で取り込めます。

  • JSONファイルを選んでインポート — スマホでも扱いやすいファイル選択方式
  • 貼り付けてインポート — AIの出力をそのままテキストエリアに貼り付け

対応フォーマットは2種類です。

共通スキーマ形式（neta-schema.json 準拠・推奨）

{
  "schemaVersion": "1.0.0",
  "datasets": [
    {
      "id": "ds-example",
      "name": "データセット名",
      "fields": [
        { "key": "theme", "label": "テーマ", "type": "text" }
      ]
    }
  ],
  "cards": [
    {
      "id": "card-001",
      "datasetId": "ds-example",
      "title": "カードタイトル",
      "summary": "概要",
      "customFields": { "theme": "睡眠" }
    }
  ]
}

旧形式（シンプルなAI出力向け）

{
  "datasetName": "データセット名",
  "cards": [
    {
      "title": "カードタイトル",
      "summary": "概要",
      "memo": "メモ",
      "customFields": { "難易度": "3" }
    }
  ]
}

取り込み時のルール

  • 同名のデータセットが既にあればそこに追加、なければ新規作成
  • 新しいカスタム項目の定義は既存データセットに自動でマージ
  • 同じIDのカードは重複して取り込まない（同じファイルを2回読んでも安全）
  • バックアップJSONを読み込むと、仕分け状態（残す / 捨てる / 保留）ごと復元
  • 多少項目が欠けていても取り込める。不正なJSONは分かりやすいエラーを表示

エクスポート

出力内容用途
全データJSON 全データセット・全カード（仕分け状態込み） バックアップ・別端末への移行
「残す」だけJSON 残すカードと、その所属データセット定義のみ quote-video-factory など下流ツールへの受け渡し
「残す」だけMarkdown 残すカードを人が読みやすい形式で整形 メモアプリへの転記、AIへの再投入

ファイル名には日付が入るため（例：neta-furui-backup-20260611.json）、過去のバックアップが上書きされません。

データの保存場所と注意

  • データはすべてブラウザの localStorage に保存されます。サーバーには何も送信されません
  • ブラウザのサイトデータを削除するとデータも消えます。定期的に「全データをJSONで保存」でバックアップしてください
  • 保存容量が不足して保存に失敗した場合は警告が表示されます

起動方法（開発者向け）

cd C:\ClaudeCode\apps\NetaChoice\neta-furui-lite
npm install
npm run dev      # 開発サーバー起動
npm run build    # 本番ビルド（dist/ に出力、PWA対応）

技術スタック：React + TypeScript + Vite + Tailwind CSS + Framer Motion + vite-plugin-pwa。 インポート/エクスポートのロジック検証は npx tsx scripts/smoke-import.ts で実行できます。

このツールが「やらないこと」

  • AIによるネタ収集（neta-dataset-builder の役割）
  • 動画・画像の生成（quote-video-factory の役割）
  • ログイン、クラウド同期、チーム共有、課金
  • 複雑な統計やダッシュボード

仕分けに集中するための道具として、意図的に機能を絞っています。
