# Claude Code 用プロンプト：ネタふるい Lite 完成版

あなたは優秀なフロントエンドエンジニア兼UI/UXデザイナーです。
以下の仕様に従って、私専用のローカルPWAアプリ「ネタふるい Lite」を完成させてください。

このプロンプトを読んだら、必要な設計・実装・テスト・動作確認まで一気に進めてください。
途中で私に細かい確認を求めず、合理的に判断して完成まで進めてください。

---

## 1. アプリの目的

「ネタふるい Lite」は、AIや自分が大量に集めたネタ・アイデア・創作素材・情報候補を、スマホで少しずつ見ながら、

- 残す
- 捨てる
- 保留

に仕分けるための、自分用のシンプルなツールです。

これは商業用の大規模アプリではありません。
自分用に、軽く、速く、使いやすく作ってください。

最重要コンセプトは以下です。

- AIやPCで大量にネタを集める
- JSONでアプリに取り込む
- スマホで空き時間に1枚ずつ見る
- 直感的に「残す / 捨てる / 保留」に分類する
- 残したものだけ後で一覧で見たり、Markdownで出力したりする

このアプリは「保存するためのメモアプリ」ではなく、  
「大量の候補をふるいにかけて、使えるものだけ残すためのアプリ」です。

## 2. 最初の前提

### 2-1. オンライン前提にしない

このアプリは、まずは完全ローカルで動作するPWAとして作ってください。

不要なもの：

- ログイン
- ユーザー登録
- サーバー
- クラウドDB
- 課金
- チーム共有
- 複雑なAI連携

必要なもの：

- ブラウザで動く
- スマホでもPCでも使える
- データはブラウザ内に保存する
- PWAとしてホーム画面に追加しやすい
- JSONインポート / JSONエクスポートでバックアップできる

### 2-2. 後からオンライン化しやすい構造にする

今はローカル保存でよいですが、将来的にFirebaseやSupabaseに移行できるように、データ構造と保存処理はできるだけ整理してください。

## 5. 扱うデータの考え方

アプリは以下の階層構造にしてください。

```txt
Dataset
  └ Card
```

Dataset は「分類対象のまとまり」です。

例：

- ミステリートリック
- 動画ネタ
- 脱出ゲームの謎
- 小説アイデア
- ゲーム案

Card は1つ1つのネタ・候補です。

例：

- 時計の針で文字を読む謎
- 医師が語る危ない生活習慣
- 占星術殺人事件
- 死刑候補が毎週絞られる学園もの

## 6. データ型

TypeScriptで以下のような型を定義してください。
必要なら実装しやすいように調整して構いません。

```ts
export type CardStatus = "unclassified" | "keep" | "discard" | "hold";

export type FieldType = "text" | "textarea" | "number" | "rating" | "select" | "multiSelect";

export interface CustomFieldDefinition {
  key: string;
  label: string;
  type: FieldType;
  showOnCard: boolean;
  options?: string[];
}

export interface Dataset {
  id: string;
  name: string;
  description: string;
  icon?: string;
  fields: CustomFieldDefinition[];
  createdAt: string;
  updatedAt: string;
}

export interface Card {
  id: string;
  datasetId: string;
  title: string;
  summary: string;
  status: CardStatus;
  memo?: string;
  customFields: Record<string, string | number | string[] | undefined>;
  reviewCount: number;
  createdAt: string;
  updatedAt: string;
  lastReviewedAt?: string;
}
```

重要：

- `status` は最初は4種類だけでよい
  - `unclassified` = 未分類
  - `keep` = 残す
  - `discard` = 捨てる
  - `hold` = 保留
- 複雑な段階選別は最初は不要
- ただし「残す」だけを後で再度仕分けできるように、一覧から状態変更できるようにする

## 10. 仕分け画面仕様

目的：

カードを1枚ずつ見て、残す / 捨てる / 保留に分類する画面です。

表示内容：

- 戻るボタン
- データセット名
- 進捗表示
  - 例：7 / 31
- 現在のカード
- 主要項目
- 操作ボタン

カードに表示する内容：

- タイトル
- 概要
- カスタム項目のうち `showOnCard: true` のもの
- メモ

操作：

- 右スワイプ：残す
- 左スワイプ：捨てる
- 上スワイプ：保留
- ボタンでも操作可能にする
  - 捨てる
  - 保留
  - 残す

重要：

- スワイプ操作だけに依存しない
- 必ずボタン操作でも分類できるようにする
- 誤操作対策として「1つ戻る」機能をつける
- 分類後は次の未分類カードを表示する
- 未分類カードがなくなったら「仕分け完了」画面を表示する
- 仕分け完了画面には「一覧を見る」「ホームへ戻る」ボタンを置く

カードUIのイメージ：

```txt
脱出ゲームの謎    7 / 31

未分類を仕分け中

[カード]
時計の針で文字を読む謎

タグ：
暗号 / 中盤向き / 難易度 3

概要：
3つの時計の針の向きから、対応する文字を読み取って答えを導く。

謎の種類：暗号
解答：トビラ
メモ：発想は良いが少し理不尽かも

[捨てる] [保留] [残す]
```

## 12. インポート機能

AIやClaudeに作らせたJSONを貼り付けてカードを追加できるようにしてください。

形式は以下のようにしてください。

```json
{
  "datasetName": "脱出ゲームの謎",
  "cards": [
    {
      "title": "時計の針で文字を読む謎",
      "summary": "3つの時計の針の向きから、対応する文字を読み取って答えを導く。",
      "memo": "発想は良いが少し理不尽かも",
      "customFields": {
        "謎の種類": "暗号",
        "解答": "トビラ",
        "難易度": "3"
      }
    }
  ]
}
```

仕様：

- JSONを貼り付けるテキストエリアを用意
- 「インポート」ボタンで読み込み
- `datasetName` が既存のデータセット名と一致する場合は、そのデータセットに追加
- 一致しない場合は新しいデータセットを作成
- カードの `status` は `unclassified`
- 不正なJSONの場合は分かりやすいエラーを表示
- インポート成功時に追加件数を表示

重要：

- AI出力をそのまま貼り付けやすいことを優先
- 厳密すぎて取り込みに失敗しやすくしない
- 多少不足項目があっても取り込めるようにする

## 13. エクスポート機能

データ消失対策と、AIに再利用するために、エクスポート機能を必ず実装してください。

必要なエクスポート：

### 13-1. JSONエクスポート

全データをJSONとしてダウンロードできるようにしてください。

用途：

- バックアップ
- 別端末への移行
- 将来のオンライン化

### 13-2. Markdownエクスポート

「残す」に分類したカードだけをMarkdownとして出力できるようにしてください。

形式例：

```md
# 残したネタ一覧

## 時計の針で文字を読む謎

データセット：脱出ゲームの謎

概要：
3つの時計の針の向きから、対応する文字を読み取って答えを導く。

メモ：
発想は良いが少し理不尽かも。

追加項目：
- 謎の種類：暗号
- 解答：トビラ
- 難易度：3
```

画面には、Markdownをコピーできるボタンも用意してください。

## 14. 保存仕様

ローカル保存を実装してください。

推奨：

- まずは localStorage でよい
- データ量が増えても拡張しやすいように、保存処理は1ファイルにまとめる
- 将来的に IndexedDB やクラウドDBに切り替えやすいようにする

保存タイミング：

- カード分類時
- カード編集時
- インポート時
- 削除時
- 状態変更時

データが消えないように注意してください。

## 16. UI/UX方針

とにかくシンプルにしてください。

デザイン方針：

- スマホで見やすい
- 白背景
- 角丸カード
- 余白多め
- 青〜紫系のアクセント
- 文字は大きめ
- ボタンは押しやすく
- ごちゃごちゃさせない
- 画面下部ナビは3つだけ

避けること：

- 複雑なダッシュボード
- 細かすぎる統計
- 多すぎる設定
- 商業アプリっぽい余計な機能
- ごちゃごちゃしたタグだらけの画面

このアプリは「自分用の軽い道具」です。

## 18. 実装上の注意

### 18-1. まず動くものを作る

見た目に凝りすぎるより、まず一通り動くことを優先してください。

ただし、最低限きれいで使いやすいUIにしてください。

### 18-2. ファイル構成を整理する

推奨構成：

```txt
src/
  components/
    BottomNav.tsx
    DatasetCard.tsx
    SortCard.tsx
    StatusBadge.tsx
    CardDetailModal.tsx
  data/
    sampleData.ts
  lib/
    storage.ts
    export.ts
    import.ts
  pages/
    HomePage.tsx
    SortPage.tsx
    ListPage.tsx
    ImportExportPage.tsx
  types.ts
  App.tsx
  main.tsx
```

ルーティングは React Router を使ってもよいですが、シンプルに state で画面切り替えしても構いません。
実装が安定する方を選んでください。

### 18-3. TypeScriptエラーを残さない

必ず TypeScript の型エラーを解消してください。

### 18-4. ビルドできる状態にする

最後に必ず以下を確認してください。

```bash
npm install
npm run dev
npm run build
```

必要なら `npm run lint` も通してください。
ただし、lint設定がない場合は無理に追加しなくてよいです。
